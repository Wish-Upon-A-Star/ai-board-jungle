## Issues Found and Resolved
- Issue: remote browsers would call their own `127.0.0.1:8000` because `dev-fastapi` hard-coded `VITE_API_BASE`.
- Resolution: default API base now uses `AI_BOARD_PUBLIC_HOST` or the first LAN IPv4 address unless the bind host is explicitly local-only.
- Issue: readiness fixture failed after adding the new network check.
- Resolution: updated compact readiness fixture and formatter fixture to require 13 checks and `PASS network config`.
- Issue: first operational probe missed Vite readiness even though logs showed it started.
- Resolution: reran with a direct startup delay and confirmed API/web HTTP responses.
- Issue: automatic LAN IP selection could pick a `169.254.*` link-local adapter address before a real LAN address.
- Resolution: prefer private LAN ranges first, skip link-local addresses where possible, and cover this with `verify:network-config`.
