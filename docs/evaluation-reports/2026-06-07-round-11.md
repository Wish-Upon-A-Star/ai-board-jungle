## Findings And Fixes
- Finding: Automations had `interval_minutes`, but the app only supported manual runs.
- Fix: Added `POST /api/automations/scheduler/tick`, which finds due ACTIVE automations by `last_run_at + interval_minutes` and runs them.
- Finding: Scheduled execution still needed the same duplicate-call protection as manual execution.
- Fix: Scheduler execution uses the same fingerprint concept and records unchanged inputs as `skipped`.
- Finding: Scheduler authorization needed to preserve user privacy.
- Fix: Normal users tick only their own due automations; admin users can tick all due automations.
- Finding: There was no visible way to test the scheduler from the UI.
- Fix: Added a `Scheduler tick` button and result display through the existing API result panel.
