# 2026-06-12 Round 280 Evaluation

## Focus
- Expose webhook setup readiness as an API and show it in the automation setup flow.

## Changes
- Added `GET /api/webhook-readiness`.
- The endpoint returns GitHub and Notion webhook setup metadata:
  - public endpoint URL,
  - event list,
  - signature header,
  - secret env name,
  - secret configured boolean,
  - setup URL,
  - matching rule,
  - dependent automation templates.
- The endpoint never returns raw webhook secret values.
- Extended automation provider setup rows to show webhook endpoint, signature header, and secret status.
- Extended API contract verification to require webhook readiness.
- Extended automation form verification to require webhook endpoint visibility.
