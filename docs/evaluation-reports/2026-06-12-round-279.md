## Focus

## Changes
- Added automation provider dependency metadata for:
  - GitHub -> Notion,
  - Notion BOARD -> GitHub Issue,
  - Notion GANTT -> Google Calendar,
  - Figma -> Google Calendar.
- Added a provider setup checklist inside the automation readiness card.
- The checklist shows:
  - provider readiness,
  - callback URL,
  - redirect source,
  - setup hint,
  - setup console link,
  - templates that depend on that provider.
- Added provider template dependency chips to the advanced integration diagnostics cards.
- Updated `verify:automation-form-ui` so it fails unless the automation form exposes route-specific provider setup callbacks.

## Current Result
- A user selecting an automation template can now see which provider setup is required before saving/running it.
- GitHub -> Notion shows GitHub and Notion callback/setup rows.
- Figma -> Google Calendar shows Figma and Google Calendar callback/setup rows.
- The profile diagnostics area also shows which automation templates depend on each provider.

## Issues Found And Resolved
- The public automation form verifier used a broad `form input:first` selector and broke after adding callback inputs.
- Resolved by scoping the automation-name check to the basic automation section.

## Next Risks
- The UI now explains callback/webhook requirements, but actual webhook auto-registration still depends on provider APIs and granted scopes.
- A fixed Cloudflare domain or production deployment is still needed to avoid repeated OAuth callback allowlist updates when temporary tunnel URLs rotate.
