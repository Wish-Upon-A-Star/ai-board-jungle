## Issues Found And Resolved

- Issue: Runtime defaults still allowed SQLite as the normal app database.
  Resolution: Added shared PostgreSQL environment helpers and changed runtime/serve/verify scripts to use PostgreSQL by default.
- Issue: Verification could interrupt the already shared server on ports 3000/8000.
  Resolution: Removed destructive local server cleanup from managed checks and moved them to isolated ports.
- Issue: README and checklist still told users SQLite was the local default.
  Resolution: Rewrote the database sections around PostgreSQL-first operation and added `verify:postgres`.
- Issue: Notion automation writes did not backfill each collected commit/issue/PR as a separate item.
  Resolution: Notion writes now loop through newly saved external sources and include source type, title, URL, next action, and extracted text.
- Issue: Command-scope verification still expected the old destructive port policy.
  Resolution: Updated the command list contract to the new non-destructive verification ports.

- The app can write to Notion only after each user stores their own Notion token and target database in an integration profile.
- The existing public Cloudflare quick tunnel is suitable for small testing, but a named tunnel or hosted deployment is still needed for stable external URLs.
