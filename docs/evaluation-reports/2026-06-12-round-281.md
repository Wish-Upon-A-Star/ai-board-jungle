# 2026-06-12 Round 281 Evaluation

## Focus
- Implement GitHub webhook auto-registration through the user's GitHub integration profile.

## Changes
- Added `WebhookRegistrationIn`.
- Added `POST /api/integration-profiles/{profile_id}/github-webhook`.
- Added GitHub webhook registration helper that:
  - parses the repository from the GitHub profile Base URL,
  - uses the user's stored GitHub token,
  - builds the GitHub hook payload,
  - redacts webhook secrets from responses,
  - supports dry-run without external GitHub calls,
  - requires `WRITE LIVE` before actual registration,
  - detects an existing hook with the same endpoint before creating a new one.
- Added GitHub profile UI controls:
  - `Webhook dry-run`,
  - `WRITE LIVE` confirmation input,
  - `실제 GitHub hook 등록`.
- Extended API contract verification to cover GitHub webhook registration dry-run.
- Extended profile UI verification to require GitHub webhook controls.
