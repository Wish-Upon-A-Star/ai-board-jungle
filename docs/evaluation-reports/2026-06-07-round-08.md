## Findings And Fixes
- Finding: External RAG collect always used a hardcoded `20 x 2p` range from the UI button, so different users/profiles could not tune collection cost or coverage.
- Fix: Added `collect_limit` and `collect_pages` to integration profiles, serialized them as `collectLimit` and `collectPages`, and made `/api/integration-profiles/{profile_id}/collect` use stored values when query parameters are omitted.
- Finding: Tests only verified query-parameter collection and did not prove stored profile values reached the collector.
- Fix: Updated FastAPI tests so a profile saved with `12 x 3p` is collected without query parameters and the fake collector receives `limit=12`, `pages=3`.
- Finding: UI still displayed a fixed range even after profile-specific collection settings existed.
- Fix: Added `Collect limit`, `Collect pages`, and `Collect scope` fields to the integration profile form, and changed profile cards to display each saved range.
- Finding: README still described the UI collect button as fixed `20 x 2페이지`.
- Fix: Updated README to explain stored per-profile collection range and optional query override.
