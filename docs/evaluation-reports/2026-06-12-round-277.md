# 2026-06-12 Round 277 Evaluation

## Focus
- Make the automation setup readiness card route-aware, so the selected template highlights only the providers required by that automation path.

## Changes
- Added `detectAutomationProviderKeys` to infer required providers from:
  - template preset,
  - source/destination,
  - API provider and agent name,
  - custom connection service/API/operation/auth fields.
- The automation readiness card now prioritizes provider chips for the current route.
- Required route providers are marked with `필수`.
- The card now reports current-route readiness, such as `현재 경로: 2/2개 준비됨`, instead of only overall provider readiness.
- If the selected route is ready even though another unrelated provider is missing, the card shows the route-ready state.
- Updated `verify:automation-form-ui` to assert:
  - default GitHub -> Notion highlights GitHub and Notion,
  - Figma -> Calendar highlights Figma and Google Calendar,
  - the profile/callback action still opens the integrations panel.
