## Findings And Fixes
- Finding: Provider readiness could show Figma/Google Calendar as ready, but there was no app API to execute those writes.
- Fix: Added `/api/integration-profiles/{profile_id}/write` and `backend/app/live_writers.py` for Figma comment and Google Calendar event writes.
- Finding: Actual external writes should not happen accidentally from the UI.
- Fix: Added `dry_run=true` as the default. UI `Live write check` validates the request URL and payload without changing external services; `dry_run=false` enables actual writes for users with saved tokens.
- Finding: Custom connection profiles could be marked as Figma/Calendar candidates but were not executable.
- Fix: `execute_profile_write()` now detects `custom_connections[].service` for `figma` and `google_calendar`, using that connection's URL.
- Finding: Google Calendar IDs may include `@` or other characters that break raw URL paths.
- Fix: Calendar IDs are URL encoded before the Google Calendar API request.
- Finding: Network errors during real external writes would raise a server error.
- Fix: HTTP failures are returned as structured `status=failed` write results and recorded in the activity log.
