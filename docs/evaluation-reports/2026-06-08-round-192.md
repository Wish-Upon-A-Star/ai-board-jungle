## Review Rounds

1. Checked current server preservation requirement; existing ports 3000 and 8000 remained untouched.
2. Checked external access approach; selected single-process FastAPI plus Cloudflare Quick Tunnel to avoid split frontend/API public exposure.
3. Checked port isolation; external default port is 8130 and verification port is 8131.
4. Checked script safety; `serve-external.mjs` does not call `stopLocalServers`.
5. Checked production serving fit; built React app is served by FastAPI so users open one public URL.
6. Checked small-team capacity; README recommends PostgreSQL and extra workers only when not using SQLite.
7. Checked verification coverage; `verify:external-serve` smoke-tests the external path without opening a public tunnel.
8. Checked command scope; external verification is listed as server-required and documented as separate from 3000/8000 ownership.
9. Checked docs; README explains public URL, overrides, and temporary tunnel limitation.
10. Checked submission checklist; it now says not to stop a shared server for ad-hoc fixes before test-port verification.
11. Checked full verification integration; `verify-full` includes the external serve check after frontend build.
12. Checked remaining risk; quick tunnel is suitable for temporary 6-10 user testing but not a stable production domain.

## Issues Found And Resolved

- Issue: LAN access did not satisfy external users outside the network.
  Resolution: Added `npm run serve:external` with Cloudflare Quick Tunnel support.
- Issue: Testing external serving could accidentally interrupt current shared ports.
  Resolution: Added `npm run verify:external-serve`, which uses port 8131 and checks the script does not stop existing servers.
- Issue: 6-10 user use on SQLite could be fragile under writes.
  Resolution: Documented PostgreSQL as the recommended database for real multi-user usage and kept worker default at 1 for SQLite safety.

## Next Risks

- Add a named Cloudflare Tunnel or hosted deployment target for a stable public URL.
- Add rate limiting and stronger session/token hardening before wider public use.
- Move the active shared instance to PostgreSQL before sustained multi-user write testing.
