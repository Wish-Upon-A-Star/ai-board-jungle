## Findings And Fixes
- Finding: The activity log always loaded the latest 50 user rows, which became noisy after profile saves, RAG collection, live writes, automation runs, and shares.
- Fix: Added server-side filters for `provider`, `status`, `event_type`, `automation_task_id`, `integration_profile_id`, and bounded `limit`.
- Finding: The UI had no way to narrow the log by provider, status, event, automation, or profile.
- Fix: Added filter selects and a reset button above `Integration Activity Log`, with server re-fetch on every filter change.
- Finding: Provider options built only from the currently filtered rows would make it hard to switch to another provider after filtering.
- Fix: Added a stable provider/status/event option list while preserving the selected provider if it is custom.
- Finding: Empty filtered logs would show a blank panel.
- Fix: Added an explicit empty state.
