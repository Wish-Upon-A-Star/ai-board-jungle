## Findings And Fixes
- Finding: New integration profile tokens were stored directly in `token_value`.
- Fix: Added `enc:v1:` encrypted-at-rest storage for new tokens using `AI_BOARD_TOKEN_ENCRYPTION_SECRET`, with local fallback to `AI_BOARD_JWT_SECRET`.
- Finding: Collectors and automation plans previously treated stored token text as the usable token.
- Fix: Collectors and automation plans now use `reveal_secret()` and only report token-ready when a token can be decrypted or read as legacy plaintext.
- Finding: Users could not see whether a token was encrypted or legacy.
- Fix: API and UI expose `tokenStorage` as `encrypted`, `legacy`, or `empty`, while never returning token plaintext.
